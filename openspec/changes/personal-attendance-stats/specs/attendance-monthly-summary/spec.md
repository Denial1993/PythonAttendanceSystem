## ADDED Requirements

### Requirement: 系統必須能計算員工指定月份的出勤統計摘要
系統必須提供一個 API 端點，接收員工帳號、年份與月份作為參數，查詢該員工當月所有打卡記錄後，計算並回傳以下統計數字：出勤天數、遲到次數、遲到總分鐘數、缺卡日數、未打下班卡日數。

#### Scenario: 正常月份統計查詢
- **WHEN** 任何已登入使用者以自身 username 查詢指定年月的統計摘要
- **THEN** 系統必須回傳包含出勤天數、遲到次數、遲到總分鐘數、缺卡日數的摘要物件，以及每日的明細清單

#### Scenario: 月份無出勤記錄
- **WHEN** 查詢的月份該員工完全沒有任何打卡記錄
- **THEN** 系統必須回傳全為零的統計摘要，並回傳空的每日明細清單，不得報錯

#### Scenario: 越權查詢阻擋
- **WHEN** Role 3 員工嘗試查詢其他員工的統計摘要（帶入不屬於自己的 username）
- **THEN** 系統必須拒絕請求並回傳 403 Forbidden

### Requirement: 遲到判定必須基於固定的上班標準時間
系統必須以 **09:00** 作為標準上班時間。若員工當日 `check_in_time` 晚於 09:00，則該日計為遲到，並計算遲到分鐘數（= 實際上班時間 - 09:00 的分鐘差）。

#### Scenario: 遲到計算
- **WHEN** 員工當日 `check_in_time` 為 09:30
- **THEN** 系統必須計算遲到 30 分鐘，遲到次數 +1，遲到總分鐘數 +30

#### Scenario: 準時出勤不計遲到
- **WHEN** 員工當日 `check_in_time` 為 08:55（早於或等於 09:00）
- **THEN** 系統必須不計為遲到，遲到次數不增加

### Requirement: 系統必須在前端呈現月份統計卡片
前端 Dashboard 必須顯示「本月出勤統計」區塊，以數字卡片方式呈現出勤天數、遲到次數、遲到總分鐘數、缺卡日數，並附上每日明細可展開的清單，讓員工能看到哪天遲到、哪天缺卡。

#### Scenario: 員工登入後自動載入本月統計
- **WHEN** 員工成功登入並進入 Dashboard
- **THEN** 系統 UI 必須自動載入並顯示當前年月的出勤統計

#### Scenario: 切換月份查看歷史統計
- **WHEN** 員工從月份選擇器選擇不同的年月
- **THEN** 前端必須重新呼叫 API 並更新統計卡片的數字與明細
