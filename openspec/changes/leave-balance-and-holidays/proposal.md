## Why

為了讓打卡與假勤系統具備真正的「ERP 管理能力」，我們需要防止員工超休（透支假勤），並精準計算請假時數。這需要基於「週年制」發配休假額度，並結合「政府公家機關辦公日曆表」來自動跳過假日計算，避免把國定假日或週末算入請假天數，同時正確計算補班日。

## What Changes

1.  **新增 `Holidays` (行事曆) 資料表**：用於建立國定假日與補班日的快取，取代每次即時打 API 計算的風險與耗時。
2.  **修改 `User` (員工) 資料表**：新增 `hire_date` (到職日) 作為特休額度 (週年制) 發放的計算基準。
3.  **新增 `LeaveBalances` (假勤存摺) 資料表**：追蹤每一位員工在各個年度 (以到職日週年為期) 在不同假別的總額度 (`total_hours`) 與已使用額度 (`used_hours`)。
4.  **升級請假申請邏輯**：在申請請假時，系統會比對 `Holidays` 表扣除假日後計算實際請假「小時數」，並檢查該假別是否仍有餘額。審核通過後自動扣除餘額。

## Capabilities

### New Capabilities
- `leave-balances`: 管理特休、事假、病假等各類假別的發放額度與消耗追蹤。
- `holiday-calendar`: 紀錄並判斷國定假日與補班日，作為假勤時數計算的基礎。

### Modified Capabilities
- `leave-application-system`: 修改請假時數計算機制（扣除國定假日，計算補班日）與綁定防呆檢查（比對剩餘假勤時數）。

## Impact

- **Database**: 需新增 `holidays` 與 `leave_balances` 資料表，並對 `users` 增加 `hire_date`。
- **Leave System API**: 申請病假/事假/特休等假單前，需加入餘額檢查機制，改變了以往單純傳送開始與結束時間的機制。
- **Frontend UI**: 請假頁面需顯示當年剩餘額度，以及在無法請假時顯示額度不足的錯誤。
