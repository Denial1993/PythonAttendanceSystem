## Context
現行系統的請假流程僅具備送出表單的紀錄功能，缺乏可用額度的審查與假日跳過的邏輯。為了邁向企業級 ERP，我們需要防止員工超額請假，並正確應對補班日與週休二日的請假天數計算問題。

## Goals / Non-Goals

**Goals:**
- 在自己的資料庫內存儲台灣政府國定假日與補班日配置
- 自動跳過假日結算請假實際「小時」數
- 根據「週年制」發配員工的特休假等各類假勤額度，記錄於獨立資料表
- 在請假申請當下阻擋額度不足的操作

**Non-Goals:**
- 不做特休遞延至下一年度的複雜審核邏輯（暫以該年到期歸零為原則）
- 不做即時政府日曆 API 調用（改採人工/排程批次匯入資料庫）

## Decisions

1.  **資料表架構**: 新增 `Holidays` 表，欄位 `is_holiday` 控制是否為例假日。如果是補班日，則設為 `false`，請假時照算時數。
2.  **特休週年制**: `users` 新增 `hire_date`。`LeaveBalances` 透過 `valid_from` 和 `valid_until` 來對應週年區間，餘額過期即失效。
3.  **以小時為單位**: 為了避免請半天假或是請 2 小時造成的四捨五入誤差，所有特休天數轉換為小時（例如 `total_hours`、`used_hours`）。

## Risks / Trade-offs

1.  **假單併發的額度透支風險**: 若員工餘額剩 8 小時，但連送兩次 8 小時的請假單，若沒有 pending 狀態的凍結額度計算，可能造成審核漏洞。初期我們將在送出時與主管審核「同意」時都再次檢查總時數。
2.  **每年手動維護**: 政府日曆若未排程自動化，需要管理員每年點擊一次更新或手動維護資料表。
