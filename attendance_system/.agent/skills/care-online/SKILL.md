# Skill: Care Online 環境相容性優化

## 核心任務
當使用者啟用 `/care-online` 時，你必須針對「正式環境」與「本地開發環境」的差異進行程式碼審查與修正。

## 檢查重點
1. **環境變數判斷**：檢查前端按鈕（如 [登入]、[註冊]）是否硬編碼了本地 URL。
2. **條件式渲染**：確保在正式機環境下，若該功能尚未開放或 API 位址不同，應有相對應的錯誤處理或隱藏邏輯。
3. **API Endpoints**：確認 Ajax 或 Fetch 請求的 Origin 是否隨環境切換。

## 執行規範
> [!IMPORTANT]
> 由於要上到正式環境，請務必考量有些按鈕在本地端可以正常運作，但正式機環境可能會因為安全性、跨域 (CORS) 或配置問題而無法使用。

1. **分析流程**：先掃描專案中涉及「登入」與「註冊」的組件。
2. **提出修正**：如果發現潛在風險（例如硬編碼 `localhost`），請提出修正建議或直接修改。
3. **驗證邏輯**：確認修正後的程式碼在正式環境下具備健壯性。
